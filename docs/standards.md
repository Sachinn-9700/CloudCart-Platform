# CloudCart Engineering Standards

## Naming Convention

- Use lowercase names
- Use kebab-case
- Example:
  - auth-service
  - product-service

---

## Branch Naming

- main
- develop
- feature/<feature-name>

Example:
- feature/auth-service
- feature/monitoring

---

## Docker Image Naming

Example:
- cloudcart/auth-service:v1
- cloudcart/product-service:v1

---

## Kubernetes Naming

Examples:
- auth-deployment
- auth-service
- cloudcart-ingress

---

## YAML Standards

- Use 2-space indentation
- Keep resource names lowercase
- Add labels to all resources
